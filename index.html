📂 Encoding (符号化)

データを特定のルールで別の文字に変換する技術。鍵がないため誰でも元に戻せます。

Base64 / Base32

頻出度: ★★★

【Base64】 64種類の文字（A-Z, a-z, 0-9, +, /）で表す形式。末尾にパディング文字 = や == が付くのが特徴です。大文字・小文字が混在します。

【Base32】 32種類の文字（A-Z, 2-7）で表す形式。数字の 0, 1, 8, 9 や小文字は使われません。末尾のパディングは最大で6つの = になることがあります。

  • Base64はアルファベットの大文字と小文字が混ざっていますが、Base32は「すべて大文字」で、数字は2〜7しか使われません。これが一瞬で見分ける最大のコツです！

Hex (16進数) / Binary (2進数)

頻出度: ★★★

【Hex】 16種類の文字（0-9, a-f）で表す形式。コンピュータの生データ（バイナリ）を文字で表す時の定番です。

【Binary】 0と1だけで表す形式。コンピュータが直接理解する最も原始的なデータ表示です。

  • Hex（16進数）は、文字の中に g 以降のアルファベット（g, h, i...など）が絶対に出てきません。
  • Binary（2進数）は、文字数が「8の倍数」で並んでいることが多く、8桁ずつ区切ることでアルファベット1文字にデコードできます。

📂 Classical Cipher (古典暗号)

コンピュータ以前に使われていた、文字をずらしたり入れ替えたりする単純な暗号。

Caesar (シーザー) / Atbash (アトバシュ)

頻出度: ★★☆

【Caesar】 アルファベットを指定された数だけ後ろにずらす暗号（例：3文字ずらすと A → D）。別名「ROT（ローテート）」。13文字ずらすものは特に「ROT13」と呼ばれます。

【Atbash】 アルファベットを「逆順」に置き換える暗号（A → Z, B → Y, C → X ...）。

  • Caesarは「ずらし幅」が最大で25パターン（アルファベットの数）しかないので、総当たりで全パターン書き出せば一瞬で解けます。
  • 「ROT13」は、13文字ずらすため、暗号化するプログラムと復号するプログラムが全く同じ（13ずらしたものを、もう一度13ずらすと26ずれて元の文字に戻るため）という面白い特徴があります。

Vigenere (ヴィジュネル)

頻出度: ★★☆

【Vigenere】 シーザー暗号の進化系。1文字ずつずらす数を変えるために、「キーワード（鍵）」を使って暗号化します。

  • 暗号文の中に、同じパターンの文字の並びが繰り返し登場する場合、その間隔から「鍵の文字数」を推測することができます（カシスキー・テスト）。
  • 鍵さえ分かってしまえば、あとはシーザー暗号と同じ要領で簡単に解読可能です。

📂 Hash (ハッシュ関数・暗号学的識別)

逆算して元のデータに戻すことが不可能な「一方通行」の関数。データの改ざん検知やパスワード保存に使われます。

MD5 / SHA1 / SHA256 識別

頻出度: ★★☆

ハッシュ値は元に戻せませんが、CTFでは「ハッシュ値の文字数（桁数）」を見ることで、どのアルゴリズムで生成されたかを識別するのが基本です。

  • 【MD5】 16進数で 32文字 (128ビット)。現在では衝突攻撃が可能で安全ではありません。
  • 【SHA1】 16進数で 40文字 (160ビット)。gitのコミットIDなどでも使われています。
  • 【SHA256】 16進数で 64文字 (256ビット)。現在の標準的な暗号強度を持っています。
  • 豆知識： ハッシュ値から元の値を逆算するのは数学的に不可能ですが、「一般的なパスワードのハッシュ値」であれば、データベースサイト（CrackStationなど）で検索すると一瞬で元の文字が特定できることが多々あります。

📂 Web (ウェブ関連エンコード)

ブラウザやサーバーが、URLやHTMLの中で特殊文字を安全に伝えるための形式。

URL Decode / HTML Decode

頻出度: ★★☆

【URLデコード】 パーセントエンコーディング。URLで使えない記号や日本語を、パーセントと16進数で表します。

【HTMLデコード】 HTMLの画面上でタグ（< や >）をそのまま文字として表示するためのエスケープ処理を元に戻します。

  • URLエンコードの見分け方： 文字のいたるところに %20 や %3F のように、「% ＋ 16進数2文字」が頻出します。ちなみに %20 は「半角スペース」のことです。
  • HTMLエンコードの見分け方： &amp; や &lt;、&#123; のように、「&（アンド）から始まって ;（セミコロン）で終わる」パターンです。
